Guide d'utilisation de l'Application

Comprenez comment fonctionne notre solution médicale intégrée

Présentation de l'Application

Notre application est une solution médicale intégrée qui permet aux professionnels de santé de générer rapidement et efficacement différents types de documents médicaux : certificats médicaux, ordonnances, certificats de décès, et schémas de vaccination. L'application offre une interface conviviale et intuitive pour une gestion optimale des documents médicaux.

Fonctionnement de l'Application

Comment l'application fonctionne-t-elle ?

  • 1
    Interface utilisateur : L'application propose une interface web moderne et intuitive pour la création de différents types de documents médicaux.
  • 2
    Saisie des données : Les utilisateurs remplissent les formulaires avec les informations nécessaires pour chaque type de document.
  • 3
    Génération automatique : L'application génère automatiquement les documents selon les normes médicales en vigueur.
  • 4
    Impression et sauvegarde : Les documents peuvent être imprimés directement ou sauvegardés dans la base de données.

Conservation des Données

Pour conserver vos données, vous devez activer le serveur API :

  • 1
    Placement des fichiers : Le fichier api_simple.py doit être placé dans le dossier de votre projet.
  • 2
    Dossier database : Le dossier database doit être placé également dans le dossier de votre projet. C'est dans ce dossier que sera créé le fichier data.db contenant toutes vos données.
  • 3
    Démarrage du serveur : Pour activer la sauvegarde des données, vous pouvez :
    • Exécuter la commande : python api_simple.py dans votre terminal
    • Ou exécuter directement le fichier start_api.bat qui se trouve dans le dossier téléchargé
    Télécharger le serveur API ici
  • 4
    Activation du serveur : Une fois le serveur démarré, l'application pourra sauvegarder les certificats et ordonnances dans la base de données SQLite.

Structure des Fichiers

Voici la structure recommandée de votre projet pour que tout fonctionne correctement : 

e:/votre_projet/
├── api_simple.py          <- Serveur API (à la racine)
├── database/              <- Dossier de la base de données
│   └── data.db            <- Fichier de base de données (généré automatiquement)
└──

Informations Importantes

  • •
    Sans le serveur API : Si le serveur API n'est pas démarré, les données seront sauvegardées temporairement dans le navigateur (localStorage) mais ne seront pas persistantes.
  • •
    Sécurité des données : La base de données SQLite est locale et sécurisée. Assurez-vous de sauvegarder régulièrement le fichier data.db.
  • •
    Port du serveur : Le serveur API s'exécute sur le port 5000 par défaut. Assurez-vous que ce port est disponible sur votre système.
